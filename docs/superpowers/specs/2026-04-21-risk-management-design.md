# 风险管理模块设计方案

## 背景

当前回测系统缺少风险管理机制，止损/止盈依靠手动调仓，无法自动执行。需要增加：
1. ATR动态止损止盈
2. 移动止损
3. 仓位管理
4. 详细交易统计

## 风险管理规则

### 1. ATR动态止损止盈

**参数配置：**
| 参数 | 默认值 | 说明 |
|------|--------|------|
| ATR周期 | 14天 | 标准值 |
| 止损倍数 | 2.0×ATR | 买入价下方2个ATR |
| 止盈倍数 | 3.0×ATR | 买入价上方3个ATR |

**ATR计算：**
```
TR = max(H-L, |H-PC|, |L-PC|)
ATR = MA(TR, 14)
```

**触发逻辑：**
1. 每日检查持仓股票
2. 如果当前价 ≤ 止损线 → 触发止损（卖出）
3. 如果当前价 ≥ 止盈线 → 进入移动止损监控模式

### 2. 移动止损

**参数配置：**
| 参数 | 默认值 | 说明 |
|------|--------|------|
| 跟踪幅度 | 10% | 最高价下方10% |
| 启动条件 | 上涨5%后 | 浮盈≥5%才启动 |

**跟踪逻辑：**
```
移动止损线 = 最高价 × (1 - 跟踪幅度)

如果最高价 = 15元，移动止损线 = 15 × 0.9 = 13.5元
如果股价涨到18元，移动止损线 = 18 × 0.9 = 16.2元
如果股价跌破16.2元 → 触发止损（卖出）
```

**执行优先级：**
1. 止损（无条件触发）
2. 移动止损（只在浮盈≥5%时生效）
3. 正常持仓

### 3. 仓位管理

**规则：**
| 项目 | 默认值 | 说明 |
|------|--------|------|
| 单只股票上限 | 20% | 资金量的20% |
| 总仓位上限 | 90% | 预留10%现金 |
| 现金备用 | 10% | 应对突发下跌 |

**买入逻辑：**
```
可用资金 = min(总资金 × 90% - 已用资金, 总资金 × 20%)
买入股数 = floor(可用资金 / 当前价 / 100) × 100  # 整手
```
