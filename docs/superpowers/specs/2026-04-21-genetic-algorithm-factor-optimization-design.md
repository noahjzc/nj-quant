# 遗传算法因子权重优化设计

> **目标**: 在最大回撤 ≤ 20% 约束下，通过遗传算法优化因子权重，最大化投资组合收益率

## 1. 问题定义

### 1.1 优化目标

```
主目标: 最大化 Calmar比率 = 年化收益率 / 最大回撤
约束: 最大回撤 ≤ 20%
优化变量: 10个因子的权重 (PB, PE_TTM, PS_TTM, RSI_1, KDJ_K, MA_5, MA_20, TURNOVER, VOLUME_RATIO, AMPLITUDE)
```

### 1.2 数据范围

```
时间范围: 2019-01 至 2024-01 (5年)
调仓频率: 每周
市场: A股全市场
初始资金: 100万
持仓数量: 5只股票，等权分配
```

---

## 2. 数据划分 (Walk-Forward Analysis)

### 2.1 窗口设计

```
训练窗口: 3年 (用于GA搜索最优权重)
验证窗口: 1年 (用于早停/选择最优个体)
测试窗口: 1年 (用于最终绩效评估)

滚动步进: 3个月 (每3个月重新优化一次权重)
```

### 2.2 窗口划分示例

| 阶段  | 训练期               | 验证期               | 测试期               |
|-----|-------------------|-------------------|-------------------|
| 第1轮 | 2019-01 ~ 2022-01 | 2022-01 ~ 2023-01 | 2023-01 ~ 2024-01 |
| 第2轮 | 2019-04 ~ 2022-04 | 2022-04 ~ 2023-04 | 2023-04 ~ 2024-04 |
| ... | ...               | ...               | ...               |

### 2.3 权重聚合

每轮优化后，取**验证集上Calmar比率最高**的权重作为该轮最终权重，用于下一期实盘。

## 3. 遗传算法设计

### 3.1 染色体编码

```
染色体长度: 10 (对应10个因子)
编码方式: 实数向量 [0.0, 1.0]
约束: 权重之和 = 1.0

示例染色体:
[0.15, 0.10, 0.05, 0.15, 0.05, 0.05, 0.05, 0.10, 0.05, 0.05]
 (PB, PE_TTM, PS_TTM, RSI_1, KDJ_K, MA_5, MA_20, TURNOVER, VOL_RATIO, AMPLITUDE)
```

### 3.2 权重约束

| 约束类型    | 范围   | 原因        |
|---------|------|-----------|
| 单因子权重下限 | 0.01 | 防止因子被完全排除 |
| 单因子权重上限 | 0.40 | 防止单一因子主导  |
| 权重总和    | 1.0  | 归一化要求     |

### 3.3 GA参数设置

| 参数     | 值         | 依据            |
|--------|-----------|---------------|
| 种群大小   | 50        | 10参数 × 5，行业经验 |
| 最大代数   | 100       | 收敛速度决定        |
| 精英保留比例 | 10%       | 保留最优个体        |
| 交叉率    | 0.7       | 行业标准          |
| 变异率    | 0.05      | 维持多样性         |
| 选择算子   | 锦标赛 (k=3) | 避免早熟收敛        |

### 3.4 遗传算子

**选择**: 锦标赛选择

```
1. 随机抽取3个个体
2. 选择适应度最高的
3. 重复直到填满交配池
```

**交叉**: 模拟二进制交叉 (SBX)

```
1. 随机选择两个父代
2. 生成两个子代，混合父代向量
3. 确保子代满足权重和=1约束
```

**变异**: 高斯变异

```
1. 选择一个基因
2. 添加均值0、标准差0.1的高斯噪声
3. 重新归一化确保权重和=1
```

**精英保留**:

```
每一代中，适应度最高的10%个体直接进入下一代
不经过选择、交叉、变异
```

## 4. 适应度函数

### 4.1 核心指标

```python
def calculate_fitness(weights, data, max_drawdown_constraint=0.20):
    """
    计算适应度

    Args:
        weights: 因子权重字典
        data: 历史价格数据
        max_drawdown_constraint: 最大回撤约束

    Returns:
        fitness: Calmar比率 (无效解返回0)
    """
    # 1. 运行回测获取绩效
    result = run_backtest(weights, data)

    # 2. 计算指标
    annual_return = result['annual_return']
    max_drawdown = result['max_drawdown']

    # 3. 约束检查
    if max_drawdown > max_drawdown_constraint:
        return 0.0  # 违反约束，返回0

    # 4. 计算Calmar比率 (防止除零)
    calmar = annual_return / max(max_drawdown, 0.01)

    return calmar
```

### 4.2 数值稳定性处理

```
问题: 如果max_drawdown接近0，Calmar会变成无穷大
解决: 使用 max(max_drawdown, 0.01) 作为分母
      即假设最小回撤为1%
```

### 4.3 适应度共享 (niching)

为防止早熟收敛，加入适应度共享机制：

```
对相似个体（权重向量欧氏距离 < σ）降低其适应度
σ = 0.1 (共享半径)
```

## 5. 过拟合防御

### 5.1 多层防御体系

| 层级 | 方法           | 作用        |
|----|--------------|-----------|
| 1  | Walk-Forward | 样本外验证     |
| 2  | 权重约束         | 限制极端权重    |
| 3  | 精英限制         | 10%上限     |
| 4  | 适应度共享        | 维持种群多样性   |
| 5  | 早停机制         | 验证集无提升则停止 |

### 5.2 早停机制

```
patience = 20代
如果验证集上20代无提升，停止训练
使用验证集上最优个体
```

### 5.3 参数敏感性分析

优化完成后，进行敏感性测试：

```
1. 固定其他因子权重
2. 逐渐改变某一因子权重 [0, 0.5]
3. 观察Calmar变化
4. 判断因子重要性
```

## 9. 风险与局限

| 风险   | 描述        | 缓解措施             |
|------|-----------|------------------|
| 过拟合  | 历史最优≠未来有效 | Walk-Forward, 约束 |
| 计算成本 | GA迭代耗时长   | 并行化, 精简数据        |
| 局部最优 | GA可能陷入局部解 | 适应度共享, 增大变异率     |
| 市场变化 | 权重可能失效    | 定期重新优化           |
